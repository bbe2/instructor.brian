K Nearest Neighbor (KNN)

  • Motivations

    • Key ideas: data points closer to each other tend to behave similarly
    • Instance-based learning

Parameters of KNN

  • Number of neighbors (K)
    • Too large: high bias
    • Too small: high variance (overfitting)
  • Distance functions
    • Euclidean
    • Manhattan
    • Minkowski

Variance vs.Bias

  • Variance: stability of model and predictions
  • Bias: accuracy of the model/methodology
  • Overfitting: high variance, low bias
  • “No Free Lunch” theorem: trade off between variance and bias, no single algorithm will outperform other algorithms on all datasets

Properties of KNN

  • Lazy learner: no active model-building is done with training dataset
    • Vs. active learner
    • No training time, long prediction time. Too slow for an online system.
  • Sensitive towards irrelevant attributes
  • Sensitive to outliers and noisy data
    • Common property to all distance-based methods
    • Requires data preprocessing: standardization, normalization
  • Non-parametric method: no assumption of data distribution
    • Could detect highly complex nonlinear decision boundary

Ensemble Learning Motivations

  • Combine multiple classifiers together and take consensus
  • Different mechanisms to produce multiple classifiers from the same datasets
    • Bagging (Bootstrap Aggregation): create multiple samples (with replacement) from the original dataset and train multiple classifiers; reduce variance
    • Boosting: an iterative algorithm that assigns different weights to the data points to focus on those tougher cases: reduce bias
    • Random Forest: train large number of decision stumps (smaller decision trees with less nodes) and use committee majority voting to decide the final classification

Model Parameters

  • learning rate
  • depth of trees
  • mtry for random forest: optional integer for number of features to randomly select at each split
  • ntree: number of classifiers

Algorithms: Random Forest

  • Produce a distribution of simple ML models on subsets of the original data

  • Combine the distribution into one “aggregated” model

  • Train
    1. Pick \(k\) features from the dataset with \(m\) features (\(k \ll m\))
    2. Among \(k\) features, pick the one that maximize information gain (or gain ratio) and use it as the node to further split the data
    3. Repeat step 2 to derive individual decision tree
    4. Repeat step 1 - 3 to generate \(n\) number of trees
  • Predict
    1. Apply each individual tree from the \(n\) random trees to the test data point and derive the outcomes and store the \(n\) predicted outcome
    2. Calculate the weight for each predicted oucome
    3. Consider the outcome with the highest weight as the final classication/prediction for the test data point

Property of the Algorithms: Pros and Cons

  • Pros
    • Perform well on a wide variety of tasks
    • Some ensemble methods such as random forest are robust to overfitting
  • Cons
    • Interpretability issue
    • Might be sensitive to outliers: boosting
    • Higher computational cost

Support Vector Machine (SVM) Motivations

  • Work with both linearly separable and linearly non-separable cases
  • Maximize margin between categories
  • Margins are only decided by the closest data points (support vectors)
    • SVM is robust for the outliers
    • Distance between data points and decision boundary indicates confidence of prediction

Kernel Tricks

  • Deal with classification that requires multiple (and nonlinear boundaries)
  • Ideas: apply mathematical functions (kernels) to project linearly unseparable data points into higher dimensional space so that a hyperplane could be found to separate different classes
  • Non-linear kernels
    • Radial basis function (RBF) \[ K(X_m, X_n) = exp(-\frac{||X_m - X_n||^2}{2\sigma^2}) \]
    • Polynomial kernel \[ K(X_m, x_n) = (X_mX_n + c)^d \]
    • Sigmoid kernel \[ K(X_m, X_n) = tanh(\kappaX_mX_n-\delta) \]

Model Parameters

  • C (Cost): used for regularization, penalty associated with misclassification
    • Soft margin classification: allow a small numbr of misclassifications
    • Hyperparameter controling penalizing use of slack variables \(\xi\)
    • Higher C, lower bias but risk of overfitting; higher complexiity of decision boundary; higher number of support vectors
    • Lower C, higher bias but lower variance
  • Kernel function specific parameters
    • \(\gamma\) for RBF: contols the tradeoff between error due to bias and variance
      • \(k(X_m, X_n) = exp(-\gamma||X_m-X_n||^2)\)
      • Higher \(\gamma\) means higher risk of overfitting; lower \(\gamma\) means higher bias; moves in the same direction as C
    • Degree for Polynomial kernel
    • Scale for Polynomial kernel

Properties of SVM

  • Pros
    • Excellent performance for a wide variety of tasks because it can deal with nonlinear decision boundary
    • Provide confidence estimate (probability transformed from distance by properly mathematics functions)
    • Robust to noisy data
    • Flexible with data representation: numerical and categorical attributes
  • Cons
    • Lack of interpretability. Black-box model
    • Risk of choosing wrong kernel function
